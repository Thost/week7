What did you think?

I first tried the Odessey.js storymap application because it seemed appropriate for the Tour de France story map that I was envisioning. Marker points were easy to create but I had trouble getting polylines between the stages to display properly (or at all).

So I decided to try the knightlab storymap and stick with start/end point markers. It worked GREAT! I was able to quickly put together slides with city locations (with the handy google maps search box embeded) as well as text and photos. It was exetremely user-friendly. There are two tabs for 'preview' and 'edit' of the storymap which allowed quick checks to make sure the map was displaying as intended. All of the text and images were from http://www.letour.com/le-tour/2016/us/

To improve, I would try to create polylines for each stage route, this would take more than point data and probably one of the more sophisticated storymap program.

Hope you enjoyed zooming around France!
